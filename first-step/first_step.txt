# git init

The `git init` command is used to **create a new Git repository**. It is the very first command you run when you want to start version control for a project.

---

## What Does `git init` Do?

When you run `git init`, Git:

* Creates a hidden `.git` directory
* Initializes an empty repository
* Prepares the project for tracking changes

The `.git` directory stores:

* Commit history
* Branch information
* Configuration files
* Staging area (index)

Without this directory, Git cannot function.

---

## Basic Syntax

```bash
git init
```

Run this command **inside your project folder**.

---

## Example

```bash
mkdir my-project
cd my-project
git init
```

Output:

```
Initialized empty Git repository in /my-project/.git/
```

Now the folder is a Git repository.

---

## Initial Branch Creation

* By default, Git creates a branch called `main` (or `master` in older versions)
* You can set the default branch name globally:

```bash
git config --global init.defaultBranch main
```

---

## Checking Repository Status After Init

```bash
git status
```

This shows:

* Current branch
* Untracked files

---

## Reinitializing a Repository

Running `git init` again:

```bash
git init
```

* Does NOT delete history
* Updates Git configuration if needed

Safe to run multiple times.

---

## When to Use `git init`

* Starting a new project
* Adding version control to an existing project
* Local-only projects before pushing to remote

---

## Common Mistakes

❌ Running `git init` inside another Git repository (nested repos)

✔ Always check:

```bash
git status
```

---

## Summary

* `git init` creates a new Git repository
* Initializes the `.git` directory
* First step in any Git workflow
* Required before using other Git commands

`git init` is the foundation of Git version control.
